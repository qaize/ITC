1. HTML Form

Form adalah sebuah html element yang digunakan khusus untuk mengirim http response dari client-side ke server-side dalam bentuk data yang di-input oleh user dalam form-element lalu di submit.

Form element adalah semua element yang biasa kalian lihat di UI yang digunakan untuk meng-input data, seperti text box, drop-down, radio button, check box dan lain sebagainya.

Seluruh form element ini bisa disubmit dan dikirim ke server melalui satu container form. Perhatikan code html yang tertulis di balik container di bawah ini. Dibawah ini bisa dilihat bahwa terdapat 3 html attributes, yaitu: action, name dan target.

  1. Action adalah sebuah internal link yang menjelaskan kemana form ini akan disubmit, dalam hal di bawah ini contohnya adalah menuju ke satu file php (walaupun tidak ada file tersebut). Tapi yang jelas bisa dikatakan kalau action attribute akan mengarahkan ke satu tempat di server-side yang siap menerima data dari form ini.
  2. Setiap form atau form element biasanya memiliki attribute name, yang memudahkan javascript atau server-side untuk mengidentifikasi element mana yang berhubungan dengan field yang dimaksudkan.
  3. Target adalah redirect attribute, dimana menyatakan kemana halaman akan direquest atau dibawa setelah submission selesai. _self artinya bahwa halaman tidak akan pergi kemana pun. Apabila _blank artinya akan dibuka sebuah tab baru.

Dan masih banyak lainnya, tetapi kita tidak akan terlalu dalam mempelajari ini dulu, karena saat ini kita fokus untuk belajar client-side dan pembuatan web application UI. Lebihnya mengenai form akan dijelaskan di pelajaran server-side.

1.1.1. Text Box

Simplenya digunakan untuk membuat suatu media untuk meng-input plain text. Text box diciptakan dengan menggunakan input element. Walaupun di kebanyakan browser anda bisa menyiptakan textbox tanpa menggunakan attribute type="text", tapi jangan pernah lakukan ini! Karena di server-side ini bisa menjadi masalah pada browser dengan versi tertentu.

Seandainya ada yang berhasil meng-input value lewat javascript sekalipun, value dari input ini tidak akan diterima di server side sama sekali.

1.1.2. Label

Kalian akan sering melihat label tag pada form, lalu akan banyak pertanyaan, "Mengapa kita harus menggunakan label tag? Kenapa tidak menggunakan plain span saja?". Label memiliki keistimewaan yang berbeda dengan span biasa, yaitu label memiliki attribute for. For digunakan untuk meng-kaitkan label dengan input element form yang bersangkutan dengan menggunakan ID yang pas, sehingga kita bisa mengakses atau bisa mem-fokus sebuah input hanya dengan meng-click labelnya.

Note!: Mungkin sebagian dari kalian berpikir, untuk apa kemampuan label ini? Kemampuan label for biasanya digunakan untuk keperluan styling atau design accessibility, dimana user tidak mampu click form element yang existing karena terlalu kecil atau alasan apapun. Nanti kita akan bahas lebih lanjut pada form element lainnya.

1.1.3. Check Box

Kalian semua pasti sudah familiar dengan check box, check box adalah form element dimana input yang diterimanya bisa lebih dari satu dalam bentuk collection. Walaupun pilihannya dalam bentuk choice, checkbox tetap menggunakan string value dan tetap menggunakan input tag sama seperti pada textbox, hanya saja type attributenya checkbox.

Agar server-side tau kalau beberapa checkbox ini adalah satu kesatuan value, kalian bisa menuliskan nama yang sama untuk setiap checkbox yang ada.

Kendaraan yang anda punya:

Element pada component seperti checkbox tidak bisa kalian style dengan menggunakan CSS begitu saja, karena style yang di dapat berasal dari browser atau OS nya. Silahkan coba buka di browser atau di OS lain, bentuk kotak checkboxnya bisa berbeda.

Tetapi ada cara untuk mengakalinya, kita bisa memanfaatkan checked pseudo class, + sibling selector dan kemampuan label untuk ini. Silahkan check CSS yang dibuatkan untuk check box di bawah ini.

1.1.5. Drop Down/ Select Option

Drop down/ Select option digunakan untuk membuat pilihan, dimana seluruh pilihannya bisa berasal dari data di dalam database atau bisa juga dibuatkan oleh front-end developer.

Keuntungan daripada drop down daripada radio button adalah, kita bisa membuat pilihan drop down sebanyak yang kita mau tanpa harus membuat UI menjadi kacau.

Bisa kalian perhatikan pada contoh code di bawah ini, drop down dibuat oleh 2 macam component html element, yaitu: select dan option. Select digunakan sebagai alat input yang terhubung dengan server-side, sedangkan option digunakan untuk membuat satu-persatu pilihannya.

Attribute name di-implementasi pada select tag, dan value masing-masing diletakan pada setiap option tag.

Kita bisa juga menggunakan option group atau opt-group untuk mengelompokan option-option yang ada. Kita bisa membuat default selected pilihan dengan menggunakan attribute selected.

1.1.6. Text Area

Text area adalah sebuah textbox yang cocok untuk menampung text yang sangat panjang. By default text area memiliki kemampuan untuk di resize ukurannya. Pada contoh di bawah ini, saya menggunakan css property resize untuk mematikan fungsi tersebut, sehingga ukuran dari text area tidak bisa di drag dan di resize.

Text area berbeda dengan kebanyakan input element, text area tidak menggunakan input tag, coba check source code yang ada.

1.1.7. Field Set dan Legend

Field Set dan Legend digunakan untuk membuat satu pengelompokan dengan judulnya. Fieldset tidak ada fungsinya dari sisi submission, tapi sangat berguna sebagai element komunikasi, terutama apabila ada beberapa element yang sifatnya satu sama lain, misalnya ada 2 textbox nama pertama, tapi yang pertama adalah nama pertama dari ibu, yang satunya lagi adalah nama pertama dari anak.

1.4. Button Elements

Satu element dari form lagi yang tidak kalah pentingnya adalah buttons atau tombol-tombol yang bisa kita gunakan untuk berbagai keperluan. Dasarnya button dibedakan menjadi 3 macam, normal button, reset button dan submit button.

Secara tag, button bisa dibedakan menjadi 2 macam, membuat button dengan menggunakan input tag atau dengan button tag. Disarankan untuk menggunakan button tag, karena lebih baru dan memiliki kelebihan untuk membuat open tag dan close tag.

1.4.1. Input Type Button

Berikut ini adalah cara membuat input type button, di sini jelas kita tidak bisa memasukan element apapun ke dalam button ini karena input tag adalah tipe single tag. Caption pada button hanya bisa text yang di-isi dari value. Button pada umumnya digunakan untuk mengaktifkan event-listener pada javascript.

1.4.2. Input Type Reset

Berikut ini adalah cara membuat input type reset, sama persis dengan input type button, hanya saja input type reset memiliki kemampuan untuk menghapus seluruh value dari form element pada form yang bersangkutan. Silahkan di coba.

1.4.3. Input Type Submit

Berikut ini adalah cara membuat input type submit, sama persis dengan input type button dan type reset, hanya saja input type submit memiliki kemampuan untuk mengirim seluruh nilai/value yang di input pada form ini ke server-side, agar nantinya akan diterima server-side language.

1.4.4. Button Type Button

Button type button sama seperti input type button secara fungsinya, hanya saja button type button adalah html tag dengan open dan close tag, itu artinya kita bisa masukan element tag apapun di dalamnya, termasuk sebuah paragraph, atau sebuah image.

2. HTML Validation

Pada bagian kedua ini kita akan melihat cara-cara melakukan validasi lewat html. Validasi dapat melalui bermacam-macam cara, tetapi pada web application semuanya bisa dibedakan menjadi 2 categori, yaitu client-side validation dan server-side validation.

Client-side validation dilakukan lewat HTML dan JS di browser kalian, sedangkan server-side validation dilakukan oleh programming language yang di implement di server anda. Caranya pun berbeda-beda, tergantung pada framework yang kalian gunakan.

Pada contoh di bawah ini kita akan melakukan client-side validation dengan menggunakan HTML tanpa menggunakan framework atau library apapun. Ini artinya kita akan mencoba validasi yang paling mendasar dan paling manual pada client-side.

Note: Style pada validation message berbeda untuk setiap browser atau setiap OS, HTML validation message tidak bisa kalian style dengan menggunakan CSS karena itu semua sudah bagian dari system.

2.1. Required

Required adalah validasi dengan menggunakan attribute tanpa value "required". Validasi ini digunakan untuk mengharuskan field ini di isi, tidak boleh kosong.

Note: Perhatikan, pada contoh ini ada 2 required input, tetapi apabila anda langsung click submit, maka validation messagenya, hanya yang first name saja. Ini semua karena validation pada HTML jalannya satu persatu sequential sesuai urutan. Cobalah isi yang first name tapi kosongkan birth city, maka birth city akan dikenakan validation message.

2.2. Maximum Length

Maximum length adalah batas sebuah textbox menyimpan sebuah character. Walaupun masuk pada section validation, sebenarnya maximum length tidak termasuk validation karena tidak mengeluarkan validation message. Max length lebih ke arah error prevention.

2.3. Pattern

Pattern yang dibentuk oleh pola RegEx atau Regular Expression. Pattern bisa dibuat dengan code-code regex. Tetapi di pelajaran ini kita tidak akan membahas RegEx terlalu jauh. Saya hanya akan beberapa contohnya saja.

2.4. Autocomplete

Autocomplete juga bukan merupakan validation, kita bisa menggunakan attribute autocomplete="off" untuk menghentikan feature autocomplete dari browser dimana browser bisa mengingat text yang pernah kita submit.

Seluruh feature bisa kita gunakan sekaligus dan kita kombinasikan.
