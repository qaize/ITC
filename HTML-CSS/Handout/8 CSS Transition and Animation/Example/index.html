Transition

Transition adalah perubahan beberapa value dari css property dari satu state ke state lain, yang di trigger oleh satu event atau interaksi.

Contoh di atas memperlihatkan kalau pada saat kita tidak hover, maka transisi tidak akan berlaku pada perubahannya ke asal. Kita harus meng-aplikasikannya 2 arah, saat di hover dan saat kembali ke kondisi semula.

Kita bisa melakukan transition lebih dari satu property sekaligus dengan waktu yang berbeda-beda. (Note!: Perhatikan, pada saat halaman di render, warna pada block di bawah ini baru muncul. Itu adalah salah satu perbedaan dari transisi untuk warna. Pada saat halaman di render pertamakali, transisi warna sudah dimulai.)

Beberapa hal property yang bisa dikombinasikan dengan transition adalah timing dan delay. Timing mengatur akselerasi (percepatan perubahan) dari transisi, sedangkan delay adalah waktu tunggu sebelum transisi dilakukan.

Animation

Lain dengan transition, animation tidak hanya membuat perubahan dari satu state ke state lainnya yang harus di trigger oleh satu event atau interaksi. Animasi adalah perubahan dari yang terdiri dari banyak state yang di sebut key frames (istilah pada animasi tradisional), dimana perubahan bisa terjadi dari satu key frame ke key frame kedua, ketiga, dan lanjut terus sesuai dengan keinginan animatornya. Dan animasi pada css tidak perlu di trigger oleh satu event.

Contoh 2 key frames animation. 2 key frames animation bisa menggunakan indikasi from dan to untuk mengidentifikasikannya.

Contoh multiple key frames animation. Multiple key frame animation menggunakan percentages sebagai indikasi dari progress animasinya.

Seperti pada transition, kita juga bisa mengatur delay dan timing

Kita bisa mengatur jumlah iteration menjadi infinite. Ya, jadi animasi tidak akan berhenti pada halaman ini.

Kita bisa membuat animasi memiliki jalan yang terbalik (reverse), sehingga animasi akan berjalan mundur. Atau kita juga bisa mengaturnya sebagai alternate untuk membuatnya bulak-balik.

Fill mode adalah property yang mengatur dimana css property harus di aplikasikan pada saat animasi sedang tidak dijalankan. Secara default, element akan menunjukan property pada selectornya sendiri saat tidak ada animasi, fill-mode bisa mengatur untuk membuat salah satu key frame menjadi tempat pemberhentian dari animasi ketika animasi sedang tidak dijalankan (tidak akan ada pengaruh pada infinite animation).
